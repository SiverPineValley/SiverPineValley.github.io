이번에도 합 배열을 이용한 문제이며, 나머지 연산의 분배 법칙을 사용해야 하는 문제였다.

나머지 연산의 분배법칙이란, $(A+B) % C = (A % C + B % C) % C$ 와 같이 나머지 연산에 대해서도 분배 법칙을 사용할 수 있다는 것이다.

각 숫자를 입력받고 합 배열을 구할 때 나머지를 해두면 나머지 연산을 한 효과를 얻을 수 있다.

따라서 각 숫자를 입력받고, 합 배열을 만들면서 나머지 연산을 해두는 식으로 합 배열을 만들기로 했다.

이제 특정 구간의 합 배열을 구해야 했는데, 두 합배열의 차가 특정 구간의 합 배열을 타나낸다고 생각했다. 즉, $(S[j] - S[i])$는 원본 리스트 i + 1 ~ j 까지의 합이게 된다.

여기까지는 내가 직접 생각해냈지만, 이후부터가 문제였는데 이 구간의 나머지 합을 어떻게 효율적으로 구할 수 있을지가 문제였다. 처음에는 이차원 배열을 만들어야 하나 싶었다. 하지만 최대 입력 값의 개수 ($10^6$)를 고려하면, 최대 10억개의 연산이 필요할 수 있으므로 이는 $Θ(n^2)$의 시간 복잡도를 나타내게 된다. 따라서 이는 답이 아니었다.

결국 풀이 도움을 받아서 해결을 하였는데, 풀이에 따르면 이 구간합의 차에도 나머지 연산 분배 법칙을 적용하는 방식을 사용하였다. 만약, S[j] % M의 값과 S[i] % M의 값이 같다면 해당 구간이 나머지 없이 딱 나눠진다는 의미였다.

</br>

#### 2\. 나누어 떨어지는 구간 개수 구하기

구간 합은 두 합 배열 원소의 차로 구할 수 있다. 위 예시에서는, S[3] - S[0] = 1 - 1 = 0 이므로, 2번째 원소 ~ 4번째 원소 간 합은 나누어 떨어진다는 의미이다. 실제로 2 + 3 + 1 = 6 이고, 6은 3으로 나누어 떨어진다.

이 예시를 사용하면 특정 구간 합 배열의 차가 0이 되면 나누어 떨어진다는 의미이다. 따라서 합 배열의 각 나머지 값의 개수를 카운트 한 뒤, 두 가지를 뽑는 경우의 수를 각각 구하면 풀이가 가능하다.

</br>

##### 2-1\. 합 배열에서 원소 값이 0인 경우

##### 2-2\. 합 배열의 구간 합의 차가 0인 경우

이제 합 배열에서 원소 값의 카운트를 구해본다. 1 => 2, 0 => 3 이다. 즉, 0의 개수 3개중 2개를 뽑는 경우의 수와 1의 개수 2개중 2개를 뽑는 경우의 수를 모두 더해주면 된다.

$$_{3}C_2 = 3 * 2 / 2 = 3$$

</br>

$$_{2}C_2 = 2 * 1 / 2 = 1$$

</br>

<div style="text-align: center">정답 = 3 + 3 + 1 = 7</div>

#### 3\. 수도 코드

> 1\. N, M 입력 받고, 배열 입력 받기  
> 2\. N 크기로 합 배열 S 정의하고, M 크기로 카운트 배열 C 정의
> 3\. 기존 배열 순회하면서 합 배열 만들고, C의 카운트 구하기
> 4\. C를 순회하면서 경우의 수 구하기

이를 구현하면 다음과 같다.

```
"""
    [10986] 나머지 합 구하기
    링크: https://www.acmicpc.net/problem/10986
    카테고리: 자료구조 > 리스트
    날짜: 2026.03.08 20:30PM
    걸린 시간:
"""
import sys
input = sys.stdin.readline

N, M = map(int, input().split())
numbers = list(map(int, input().split()))
S = [0] * N
C = [0] * M
answer = 0

for i in range(N):
    S[i] = (S[i-1] + numbers[i]) % M
    C[S[i]] += 1
    if S[i] == 0:
        answer += 1

for c in C:
    answer += (c * (c - 1) / 2)

print(int(answer))
```
