- **인스턴스 > 작업 > 이미지 생성**
- **이미지 > AMI**
- EC2 인스턴스의 커스텀화된 이미지를 의미한다.
	- 원하는 소프트웨어, 설정, OS, 모니터링 툴을 설치해놓을 수 있다.
	- AMI를 따로 설정하면 부팅 및 설정에 드는 시간을 줄일 수 있다. AMI가 이런 사전 준비 사항들을 패키지화해놓는다.
	- AMI는 **특정 리전에서 빌드**되는데, 직접적으로 다른 리전에서 동작은 불가능하다. 다른 리전으로 복사할 수 있다.
	- AMI에는 다음과 같은 종류가 있다.
		- Public AMI: AWS가 제공하는 공식 AMI
		- 유저가 스스로 생성한 AMI
		- AWS 마켓플레이스의 AMI
- 이미 생성된 인스턴스의 타입은 변경 불가능하다.

- AMI의 프로세싱 절차는 다음과 같다.
	- EC2 인스턴스를 시작하고 **커스터마이징**한다.
	- 데이터 무결성을 위해 **인스턴스를 중단**한다.
	- **AMI를 빌드**한다. 동시에 **EBS 스냅샷도 생성**한다.
	- 다른 AMI에서 인스턴스를 실행할 수 있다.

</br>

## 4-3-1) 시작 구성 vs 시작 템플릿

| 항목                 | 시작 구성         | 시작 템플릿     |
| ------------------ | ------------- | ---------- |
| 수정 가능 여부           | 불가 (새로 생성 필요) | 가능 (버전 관리) |
| Spot 인스턴스 지원       | ❌             | ✅          |
| 버전 관리              | ❌             | ✅          |
| 네트워크 인터페이스         | 제한적           | 여러 개 지원    |
| 유연성                | 낮음            | 높음         |
| Auto Scaling 그룹 통합 | 가능            | ✅ (권장)     |
