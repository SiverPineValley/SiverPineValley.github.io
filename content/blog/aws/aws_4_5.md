- EBS 볼륨은 6가지 타입이 존재한다.
	- **gp2 / gp3 (SSD)**: 범용 목적의 SSD 볼륨으로, 다양한 업무에 적합한 적당한 가격과 성능을 제공한다.
	- **io / io2 Block Express (SSD)**: 고성능 SSD 볼륨으로 미션 크리티컬, 저지연, 고성능 작업에 어울린다.
	- **st1 (HDD)**: 저비용의 대용량 HDD 볼륨으로, 자주 액세스되고 처리량이 많은 작업을 위해 설계되었다.
	- **sc1 (HDD)**: 가장 저렴한 HDD 볼륨으로, 액세스 빈도가 낮은 작업을 위해 설계되었다.
- EBS 볼륨은 사이즈, 처리량, IOPS(초당 입출력 처리량)에 따라 분류된다.
- EC2 인스턴스에서는 부트 볼륨으로 gp2/gp3/io1/io2 Block Express만 사용이 가능하다.

</br>

## 4-5-1) gp2 / gp3 (SSD)

- 비용 효율적이며, 지연이 적다.
- 시스템 부팅 볼륨, 가상 데스크톱, 개발 및 테스트 환경으로 사용 가능하다.
- 1GiB - 16TiB
- **gp3**: 
	- 3,000 IOPS와 125 MiB/s 의 처리량을 보장한다.
	- 16,000 IOPS와 1000 MiB/s 까지 성능을 독립적으로 올릴 수 있다.
- **gp2**:
	- 작은 gp2 볼륨은 3,000 IOPS까지 버스트 성능을 제공한다.
	- 볼륨 사이즈와 IOPS는 연동되어 있어, 최대 16,000 IOPS까지 제공된다.
	- GB마다 3 IOPS 씩 성능이 좋아지며, 5,334 GB 가 되야 최대 IOPS가 제공된다.

</br>

## 4-5-2) Provisioned IOPS (PIOPS) SSD

- 지속적인 IOPS 성능이 요구되는 중요 업무 애플리케이션에서 사용된다. 혹은 16,000 IOPS 이상이 요구될 때 사용된다.
- Database 업무에 적합하다 (스토리지 성능과 일관성에 민감한 경우)
-  **io1 (4 GiB - 16 TiB)**
	- 최대 PIOPS: Nitro EC2 인스턴스는 64,000, 다른 인스턴스는 32,000
	- 스토리지 크기와 별도로 PIOPS를 늘릴 수 있다.
- **io2 Block Express (4 GiB - 64 TiB)**
	- 서브 밀리초 단위의 저지연이 발생한다.
	- 최대 PIOPS: 최대 256,000이 제공되는데, IOPS:GiB 비율이 1000:1 이다.
- **EBS 다중 연결 기능을 제공**한다.

</br>

## 4-5-3) Hard Disk Drives (HDD)

- 부트 볼륨이 될 수 없다.
- 125 GiB - 16 TiB
- **st1 (처리량 최적화 HDD)**
	- 빅 데이터, 데이터 웨어하우스, 로그 프로세싱
	- 최대 처리량 500 MiB/s - 최대 IOPS 500
- **sc1 (Cold HDD)**
	- 자주 액세스되지 않은 데이터
	- 저비용으로 처리되어야 하는 경우 (주로 아카이빙용)
	- 최대 처리량 250 MiB/s - 최대 IOPS 250
