- EC2(Elastic Compute Cloud)는 AWS에서 제공하는 가장 인기있는 서비스 중 하나로, 여러 가지 자원 및 서비스를 제공한다.
	- 가상 머신 빌려주기 (EC2)
	- 가상 드라이브에 데이터 저장 (EBS)
	- 여러 머신에 부하 분산 (ELB)
	- 오토 스케일링 그룹을 사용하여 서비스 스케일링 (ASG)
- EC2의 사이징 & 설정 옵션들은 다음과 같다.
	- OS: Linux, Mac OS, Windows
	- CPU: Computing Power & Cores
	- Memory
	- Storage:
		- 네트워크 사용 (EBS & EFS)
		- 하드웨어 (EC2 Instance Store)
	- 네트워크 카드: 네트워크 속도, 공용 IP 주소
	- 방화벽 정책
		- 보안 그룹
	- **부트스트랩 스크립트: EC2 유저 정보**
		- EC2 유저 스크립트를 사용하여 인스턴스를 부트스트랩 하는 것이 가능하다.
		- 즉, 머신이 시작할 때 한번 동작하는데 사용할 스크립트를 미리 작성해놓을 수 있다.
			- 업데이트 설치
			- 소프트웨어 설치
			- 인터넷에서 일반 파일 다운로드
		- EC2 유저 데이터 스크립트는 루트 유저로만 동작 가능하다.

</br>

## 2-2-1) EC2 Instance Types

- EC2에는 다양한 사용법에 따라 최적화된 여러 종류의 인스턴스 타입이 존재한다.
- AWS에서는 다음과 같은 네이밍 관습을 갖는다.

- m: 인스턴스 클래스
- 5: 세대 (시간에 따라 향상됨)
- 2xlarge: 인스턴스 클래스의 사이즈 (micro, small, large, 2xlarge, 4xlarge)

</br>

### 2-2-1-1) 범용 목적

- 웹 서버나 코드 저장소와 같은 다양한 종류의 작업을 수행하는 데 적합하다.
- 컴퓨팅, 메모리, 네트워킹과 같은 모든 자원의 밸런스가 좋다.
- 이 코스에서는 범용 컴퓨팅 인스턴스로 t2.micro를 사용한다.

</br>

### 2-2-1-2) 컴퓨팅 최적화 목적

- 높은 퍼포먼스의 작업에서 요구하는 컴퓨팅 집약적인 작업을 수행하는 데 적합하다.
- 배치 프로세싱, 미디어 인코딩 변환, 고성능 웹 서버, 고성능 컴퓨팅, 과학 모델링, 머신러닝, 게이밍 서버 등
- C로 시작하는 네이밍을 가진다.

</br>

### 2-2-1-3) 메모리 최적화 목적

- 메모리 내에서 대용량의 데이터 처리 작업을 수행하는데 적합하다.
- 고성능의 관계형/비관계형 데이터베이스, 분산 웹 스케일 캐시 스토어, BI에 최적화된 인-메모리 데이터베이스, 대용량의 비정형 데이터의 실시간 처리

</br>

### 2-2-1-4) 스토어 최적화 목적

- 로컬 스토리지의 대용량 데이터로의 액세스하는 작업을 수행하는데 적합하다.
- 고주파 온라인 트랜잭션 처리(OLTP) 시스템, 관계형 & 비관계형 데이터베이스, 인-메모리 데이터베이스(Redis), 데이터 웨어하우스 애플리케이션, 분산 파일 시스템
